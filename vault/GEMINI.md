## Estrutura do vault

```
01 - PROJECTS/    ← projetos ativos
02 - AREAS/       ← áreas de vida (saúde, finanças, estudos...)
03 - RESOURCES/   ← templates, referências, materiais
04 - DIARY/       ← diário / journaling
05 - MEETINGS/    ← resumos de reuniões e calls
06 - KNOWLEDGE/   ← cursos, ebooks, anotações de estudo
07 - HANDOVERS/   ← snapshots de sessões Claude
```

## Guia de contexto

Sempre que precisar de contexto, siga esta ordem:

1. `GEMINI.md` — este arquivo (identidade + projetos + estilo)
2. `01 - PROJECTS/{projeto}.md` — contexto detalhado de cada projeto
3. `07 - HANDOVERS/` — última sessão salva
4. `04 - DIARY/` — contexto do dia
